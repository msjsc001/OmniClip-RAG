# 旧 UI 全量功能与布局复刻清单

## 范围与基准

本清单基于当前旧版 Tk UI 的真实实现整理，基准源码如下：

- `omniclip_rag/gui.py`
- `omniclip_rag/config.py`
- `omniclip_rag/ui_i18n.py`

本清单的用途不是做“功能概述”，而是作为新版 Qt UI 的逐项复刻验收表。只有本清单全部通过，旧版 Tk UI 才具备退场资格。

---

## 顶层结构总览

| 层级 | 当前旧 UI 结构 |
| --- | --- |
| 根窗口 | 单窗口桌面应用，标题 `OmniClip RAG · 方寸引` |
| 顶栏 | 图标、标题、副标题、引导语、语言切换、版本徽标 |
| 主体 | 顶层 `Notebook`，分为 `查询` 与 `配置` 两个主页签 |
| 查询页 | 上方搜索区 + 下方结果与详情区，二者由纵向分隔条控制 |
| 配置页 | 左侧工作区容器，内部再含 `开始 / 设置 / UI / 检索强化 / 数据` 五个子页签 |
| 页脚 | 左侧状态文案 + 右侧结果摘要文案 |

## 一、全局窗口与公共壳层

### 1. 根窗口与启动体验

- [ ] 启动时启用 Windows 高 DPI 感知，避免系统缩放下发虚。
- [ ] 根窗口标题为 `OmniClip RAG · 方寸引`。
- [ ] 根窗口默认尺寸为 `1560x1000`。
- [ ] 根窗口最小尺寸为 `1320x860`。
- [ ] 根窗口加载 `resources/app_icon.png` 与 `resources/app_icon.ico` 作为程序图标。
- [ ] 启动后优先渲染 UI，再加载配置，再加载当前工作区状态。

### 2. 顶栏

- [ ] 顶栏左侧显示应用图标。
- [ ] 顶栏显示主标题 `title`。
- [ ] 顶栏显示副标题 `tagline`。
- [ ] 顶栏显示引导文案 `header_guide`，且会随可用宽度自动换行。
- [ ] 顶栏右侧提供语言切换下拉框，目前支持 `zh-CN` 与 `en`。
- [ ] 切换语言后会重建 UI，并刷新全部运行态文案。
- [ ] 顶栏右侧显示版本徽标，文案格式为 `Vx.y.z`。

### 3. 顶层导航

- [ ] 主体使用顶层页签切换 `查询` 与 `配置`。
- [ ] 切换主页签时，不丢失当前查询结果、配置值与日志。
- [ ] 主体内部的次级页签切换同样保留当前状态。

### 4. 页脚

- [ ] 页脚左侧持续显示 `status_var`，用于提示当前动作、成功、失败、阻塞、等待状态。
- [ ] 页脚右侧持续显示 `result_var`，用于提示查询结果数量或空状态。

### 5. 全局样式与缩放

- [ ] 旧 UI 具备浅色 / 深色 / 跟随系统三种主题逻辑。
- [ ] 旧 UI 具备 `80%-200%` 的 UI 百分比缩放能力。
- [ ] 主题切换会同步更新按钮、输入框、表格、状态条、滚动条、文本区配色。
- [ ] 缩放会同步影响字体、内边距、表格行高、按钮尺寸、文本区域内边距。

### 6. 全局公共行为

- [ ] 所有主要控件均带 tooltip 提示。
- [ ] 文件夹选择统一使用目录选择器。
- [ ] 帮助与更新按钮会先弹确认框，再打开 GitHub 仓库链接。
- [ ] 所有后台任务结束、失败、阻塞、取消都要落活动日志。
- [ ] 关闭窗口前会尝试保存当前 UI 主题、缩放、布局、配置与工作区路径。

## 二、查询主页复刻清单

### 1. 查询页整体布局

- [ ] 查询页是顶层主页签中的第一个页签。
- [ ] 查询页由纵向分隔布局构成：上方搜索卡，下方结果与详情卡。
- [ ] 上下分隔条位置可拖动。
- [ ] 上下分隔条位置会持久化并在下次启动恢复。

### 2. 搜索卡

- [ ] 搜索卡包含标题 `search_title` 与副标题 `search_subtitle`。
- [ ] 搜索卡右上角显示独立的查询状态提示条。
- [ ] 搜索卡显示查询提示文案 `query_hint`。
- [ ] 搜索输入区包含一个主查询输入框。
- [ ] 主查询输入框支持按 `Enter` 直接发起查询。
- [ ] 搜索输入区包含 `查询` 按钮。
- [ ] 搜索输入区包含 `查询并复制` 按钮。
- [ ] 搜索输入区包含 `复制上下文` 按钮。
- [ ] 搜索卡下方包含分数阈值输入框。
- [ ] 搜索卡下方包含查询条数输入框。
- [ ] 查询条数区域会显示动态推荐提示 `query_limit_hint`。
- [ ] 查询条数标签和输入框的 tooltip 会叠加动态推荐说明。

### 3. 查询状态提示条

- [ ] 空闲时显示 `没有查询任务` 风格的 idle 状态。
- [ ] 有其他后台任务占用时显示 blocked 状态，并指出阻塞任务名称。
- [ ] 热监听运行时显示 blocked 状态，并提示当前不可查询。
- [ ] 查询运行时显示 running 状态，标题含百分比。
- [ ] 查询运行时详情显示当前阶段，如准备、召回、精排、整理等。
- [ ] 查询完成后显示 done 状态，标题带完成时间 `HH:MM`。
- [ ] 查询完成后详情显示本次命中数量。
- [ ] 查询并复制完成时使用单独的完成标题文案。
- [ ] 状态条会根据 `idle / blocked / running / done` 切换颜色与边框。

### 4. 结果与详情卡

- [ ] 结果卡包含标题 `results_title` 与副标题 `results_subtitle`。
- [ ] 标题区右上角包含 `页面屏蔽规则` 按钮。
- [ ] 标题区右上角包含 `敏感过滤` 按钮。
- [ ] 工具栏包含 `全选/不全选` 上下文选择切换按钮。
- [ ] 工具栏包含 `页面排序/恢复原排序` 按钮。
- [ ] 工具栏显示页面屏蔽规则摘要。
- [ ] 工具栏显示当前上下文选中数量摘要。
- [ ] 结果卡主体使用纵向分隔布局：上方结果表格，下方详情页签。
- [ ] 结果表格与详情区之间的分隔条可拖动。
- [ ] 结果表格与详情区分隔条位置会持久化并在下次启动恢复。

### 5. 查询结果表格

- [ ] 结果表格包含 5 列：`包含`、`页面`、`原因`、`锚点`、`分数`。
- [ ] 列默认宽度分别为 `100 / 220 / 220 / 320 / 90`。
- [ ] 结果表格支持垂直滚动。
- [ ] 查询完成后默认将全部命中片段标记为已选中。
- [ ] 结果表格首列用 `[x] / [ ]` 表示该片段是否纳入上下文。
- [ ] 点击首列只切换当前片段的纳入状态，不触发整页重建。
- [ ] 点击普通行会切换当前选中片段，并刷新右侧预览。
- [ ] 查询无结果时表格为空，预览区显示 `no_results`。
- [ ] 表头点击支持排序。
- [ ] 再次点击同一表头时切换升序/降序。
- [ ] `分数` 与 `包含` 列首次点击默认按降序。
- [ ] 其他文本列首次点击默认按升序。

### 6. 页面排序逻辑

- [ ] 页面排序按钮按“页面平均分”对结果做分组排序。
- [ ] 页面分组键由 `页面标题 + 源路径` 共同决定。
- [ ] 同一页面下多个片段取平均分后参与页面排序。
- [ ] 页面平均分高的页面排在前面。
- [ ] 同分页面按原始出现顺序与标题稳定排序。
- [ ] 页面排序开启后，普通列排序状态暂时清空。
- [ ] 再次点击页面排序按钮后恢复到排序前的原始结果顺序。
- [ ] 如果用户点击任一列表表头排序，页面排序模式会退出并恢复常规列排序。

### 7. 上下文选择逻辑

- [ ] `全选/不全选` 按钮会根据当前选中数量自动切换文案。
- [ ] 若当前页所有片段均选中，再点击会全部取消。
- [ ] 若当前页未全选，再点击会全部选中。
- [ ] 每次片段选中状态变化后，上下文文本会重建。
- [ ] 每次片段选中状态变化后，摘要文案会显示 `已选 x / 共 y`。

### 8. 详情页签区

- [ ] 详情区内部包含三个页签：`片段详情`、`完整上下文`、`活动日志`。
- [ ] 任务完成后可主动切换到指定详情页签。
- [ ] 查询完成后自动切到 `完整上下文` 页签。
- [ ] 预检、错误、监控报错等流程会自动切到 `活动日志` 页签。

### 9. 片段详情页签

- [ ] 片段详情显示当前选中命中的页面名。
- [ ] 片段详情显示当前选中命中的锚点。
- [ ] 片段详情显示源文件路径。
- [ ] 片段详情显示分数。
- [ ] 片段详情显示命中原因，缺失时显示回退文案。
- [ ] 片段详情显示摘要片段文本。
- [ ] 片段详情显示完整片段文本。

### 10. 完整上下文页签

- [ ] 完整上下文由当前查询文本 + 当前选中的片段组合生成。
- [ ] 上下文生成支持 `standard` 与 `ai-collab` 两种导出模式。
- [ ] 上下文页签顶部包含按页面跳转的下拉框。
- [ ] 上下文页签顶部显示页面数量与片段数量摘要。
- [ ] 下拉框选项来自上下文文本中的页面章节解析结果。
- [ ] 选择页面后，文本区会滚动到对应页面章节起始行。
- [ ] 点击 `复制上下文` 会复制当前完整上下文文本。
- [ ] 没有可复制上下文时弹出提示框。

### 11. 活动日志页签

- [ ] 活动日志显示启动日志、配置加载、查询完成、建库进度、监控事件、错误等。
- [ ] 日志追加时不覆盖已有内容，而是按时间顺序追加到底部。
- [ ] 日志为空时显示空态文案。

### 12. 三个文本页签公共能力

- [ ] `片段详情 / 完整上下文 / 活动日志` 三个文本区均为只读文本区。
- [ ] 三个文本区均有独立垂直滚动条。
- [ ] 三个文本区底部都带文本搜索条。
- [ ] 文本搜索条包含状态提示、搜索输入框、`查找` 按钮、`下一个` 按钮。
- [ ] 输入搜索关键词后支持高亮全部命中。
- [ ] 当前命中项使用单独高亮色。
- [ ] 搜索状态提示显示 `第 x / y 个`。
- [ ] 未输入搜索词时显示空提示。
- [ ] 搜索无结果时显示未命中提示。

### 13. 页面屏蔽规则弹窗

- [ ] 查询页可打开 `页面屏蔽规则` 弹窗。
- [ ] 弹窗为独立 `Toplevel`，已打开时再次点击只激活原窗口。
- [ ] 弹窗包含标题与说明文案。
- [ ] 弹窗主体为可滚动规则列表。
- [ ] 每条规则包含 `启用勾选框 + 正则输入框 + 删除按钮`。
- [ ] 列表顶部有表头：是否启用、正则表达式。
- [ ] 支持新增规则行。
- [ ] 支持恢复默认规则。
- [ ] 支持保存规则。
- [ ] 保存时会过滤空规则。
- [ ] 保存后会刷新规则摘要。
- [ ] 若当前查询框非空且系统不忙，保存后会自动重新执行查询。

### 14. 敏感过滤弹窗

- [ ] 查询页可打开 `敏感过滤` 弹窗。
- [ ] 弹窗为独立 `Toplevel`，已打开时再次点击只激活原窗口。
- [ ] 弹窗包含标题与说明文案。
- [ ] 弹窗包含核心过滤开关。
- [ ] 弹窗包含扩展过滤开关。
- [ ] 弹窗包含自定义规则说明文字。
- [ ] 弹窗包含多行自定义规则输入框。
- [ ] 弹窗支持保存。
- [ ] 保存后会更新查询过滤配置。
- [ ] 若当前查询框非空且系统不忙，保存后会自动重新执行查询。

- [ ] 未勾选任何清理项时，点击清理会弹提示框。
- [ ] 清理前会弹确认框。
- [ ] 清理完成后会重置四个清理勾选项。

## 四、后台任务、监控与状态流

### 1. 统一任务阻塞规则

- [ ] UI 正在执行后台任务时，不允许再次启动其他后台任务。
- [ ] 热监听运行时，不允许启动普通任务和查询。
- [ ] 必须先停止热监听，才能执行被监听阻塞的动作。

### 2. 预检流程

- [ ] 可独立执行空间预检。
- [ ] 预检完成后更新预检摘要。
- [ ] 预检完成后写入活动日志。
- [ ] 预检完成后自动切到查询页的日志页签。

### 3. 模型下载流程

- [ ] 若模型已就绪，点击下载模型会给出“已就绪”提示。
- [ ] 若模型未就绪，先弹出自动下载 / 手动下载 / 取消三选一对话框。
- [ ] 若选择手动下载，会弹出包含镜像地址、官方地址、本地目标目录、预估大小的说明框。
- [ ] 若开启 `仅本地模型` 且模型不存在，会额外确认是否允许联网下载。
- [ ] 下载完成后刷新模型状态芯片与日志。

### 4. reranker 下载流程

- [ ] reranker 下载流程与模型下载流程类似，但使用 reranker 独立文案和目录。
- [ ] 若 reranker 已存在，点击会提示已就绪。
- [ ] 下载完成后刷新 reranker 状态摘要与日志。

### 5. 重建索引流程

- [ ] 若检测到未完成的 pending rebuild，会先弹出继续/放弃的选择框。
- [ ] 若已有索引存在，再次重建前会弹确认框。
- [ ] 重建任务支持暂停。
- [ ] 重建任务支持继续。
- [ ] 重建任务支持取消。
- [ ] 重建过程中任务详情会随阶段变化为 indexing / rendering / vectorizing 等不同说明。
- [ ] 重建完成后刷新文件数、片段数、引用数。
- [ ] 若存在重复 block id，会在状态与日志中特别提示。

### 6. 查询流程

- [ ] 查询前会校验查询文本非空。
- [ ] 查询前会校验分数阈值为合法数字。
- [ ] 查询时若模型未就绪，会先走模型准备流程。
- [ ] 查询进度会通过后端进度事件驱动 UI 状态条与任务面板。
- [ ] 查询完成后默认全选所有命中片段。
- [ ] 查询完成后刷新结果表、上下文、预览、结果摘要、活动日志。

### 7. 刷新状态流程

- [ ] 点击刷新会重新拉取当前工作区状态快照。
- [ ] 刷新完成后更新统计、监听摘要、reranker 摘要、状态栏和日志。

### 8. 数据清理流程

- [ ] 清理动作通过后台任务执行。
- [ ] 清理完成后刷新当前工作区状态。
- [ ] 清理完成后状态栏和日志给出反馈。

### 9. 热监听流程

- [ ] 热监听启动时切换按钮文案为停止，并使用危险按钮样式。
- [ ] 热监听停止后恢复启动按钮文案与主按钮样式。
- [ ] 热监听运行时显示当前模式：watchdog 或 polling。
- [ ] 热监听更新会刷新文件数、片段数、引用数。
- [ ] 热监听事件会写入活动日志。
- [ ] 热监听报错会自动切到查询页日志页签并弹错误框。

## 七、旧 UI 的实现特征与迁移注意点

- [ ] 多数说明文字依赖运行时自动换行，Qt 版需要保持同等可读性。
- [ ] 配置页五个子页签当前是“滚动容器 + 内容面板”模式，新版不能回退成内容溢出不可滚动。
- [ ] 查询结果的核心不是表格本身，而是“结果表 + 选中片段集合 + 上下文重建 + 文本预览”这一整套联动。
- [ ] 查询状态条不是假状态，而是和 `busy / watch / latest_task_progress / query_last_completed_at` 四类运行态联动。
- [ ] 页面排序不是普通列排序，属于独立的分组排序模式，必须单独复刻。
- [ ] 热监听、重建暂停/继续/取消、恢复未完成建库，都是新版必须保留的桌面工具级能力。
